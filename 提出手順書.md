# 内定者研修 課題提出手順書（Fork → PR 提出）

## はじめに
この文書は、研修課題を GitHub を利用して提出するための手順をまとめたものです。  
以下の手順に従って、課題を Fork → 実装 → PR（Pull Request）として提出してください。

---

# 1. 課題リポジトリへアクセス
研修担当者より指定された課題用 GitHub リポジトリへアクセスしてください。

例）https://github.com/ORG/training-week1

---

# 2. リポジトリを Fork する
画面右上の **「Fork」** ボタンをクリックし、自分のアカウントにコピーします。

これにより、あなた専用の課題リポジトリが作成されます。

---

# 3. 自分の Fork したリポジトリを clone（ローカルへ取得）

```bash
git clone https://github.com/<あなたのGitHub名>/training-week1.git
cd training-week1
```

---
# 4. 指定されたファイルに実装を行う
課題内容および記述場所は、課題リポジトリの README に書かれています。

例：

- `/src/main.py`
- `/src/index.js`
- `/src/Main.java`

など、言語ごとに指定されたファイルへ実装してください。

---

# 5. 完成したら commit & push を行う

例：

```bash
git add .
git commit -m "Week1 課題提出"
git push origin main
```
---

# 6. PR（Pull Request）を作成する

1. GitHub 上のあなたのリポジトリへアクセス  
2. 画面上部に表示される **「Compare & Pull Request」** をクリック  
3. 以下のように設定します：

- **Base（提出先）**：研修担当者の課題リポジトリ `main`
- **Compare（あなた側）**：あなたのリポジトリの `main`

4. PR タイトルは以下の形式で記載してください：
WeekX-課題提出(氏名)
例：
JavaTraining-Week1-課題提出(提出太郎)

6. 本文は PR の内容を記入してください。

7. 最後に **「Create Pull Request」** を押して提出完了です。

---

# 7. レビューへの対応

提出後、PR に対してレビューコメントが届きます。  
修正が必要な場合は、ローカルで修正し、再度 push してください。

```bash
git add .
git commit -m "指摘対応 修正概要"
git push origin main
```

# 8. 提出完了

全ての指摘が解消されたら、研修担当者が PR を Close します。
これでその週の課題提出は完了です。
